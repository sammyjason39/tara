## Agent Capabilities

### Core Processing Methods

1. **processLeaveRequestSubmission()**: Processes new leave requests, emits events, notifies supervisors
2. **processLeaveRequestApproval()**: Updates balance, emits events, sends confirmation
3. **processLeaveRequestRejection()**: Emits events, notifies employee with reason

### Notification Methods

1. **notifySupervisorForApproval()**: Sends private notification to supervisor with action required
2. **sendApprovalConfirmation()**: Sends private notification to employee confirming approval
3. **sendRejectionNotification()**: Sends private notification to employee with rejection reason

### Scheduled Tasks

1. **processPendingLeaveRequests()**: Runs every minute (Cron: '*/1 * * * *')
   - Finds pending requests older than 5 minutes
   - Re-notifies supervisors if needed
   - Ensures no request is forgotten

2. **generateDailyLeaveRequestSummary()**: Runs daily at 6 PM (Cron: '0 18 * * *')
   - Counts submitted, approved, rejected requests
   - Tracks pending total
   - Emits summary event for HR dashboard

### Monitoring

1. **getHealthStatus()**: Returns agent health metrics
   - pending_requests: Current pending count
   - processed_today: Requests submitted today
   - approved_today: Requests approved today
   - rejected_today: Requests rejected today
   - events_emitted_today: Events published today
   - Returns 'healthy', 'degraded', or 'unhealthy' status

## Architecture Notes

The Leave Request Agent follows the autonomous agent architecture pattern:

- **Asynchronous Processing**: Uses `setImmediate()` to process requests without blocking
- **Event-Driven**: Emits events to Event Bus for consumption by other agents and systems
- **Self-Monitoring**: Provides health status for monitoring dashboards
- **SLA Enforcement**: Scheduled tasks ensure SLA compliance
- **Graceful Degradation**: Logs errors but doesn't fail the main operations
- **Idempotent**: Can safely re-process pending requests without side effects
