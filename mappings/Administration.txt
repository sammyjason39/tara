# Admin Module Map

## Purpose
The Admin (Administration) module serves as a centralized service desk for departmental requests, ensuring that internal needs are routed, tracked, and resolved efficiently.

## Navigation & Workflows

### 1. Request Creation
- **Path**: Dashboard → Requests → Create
- **Button**: "New Request"
- **Purpose**: To log a generic or specific administrative need (e.g., facility repair, office supplies, travel booking).
- **Actions**:
  - Input: Request title, category, priority, details.
  - Trigger: "Submit"
  - Approval: HOD (Head of Department) of the requester.
  - Result: Ticket created in the global administrative queue.

### 2. Assignment & Routing
- **Path**: Dashboard → Requests → Assign
- **Button**: "Dispatch Request"
- **Purpose**: To route the ticket to the correct department (IT, HR, Facilities, etc.).
- **Actions**:
  - Input: Assigned Department, Assigned Personnel.
  - Validation: Recipient department must be active.
  - Trigger: "Submit"
  - Escalation: Automatic alert if no response within predefined SLA (Service Level Agreement).
  - Result: Request moved to "In Progress" status and recipient notified.

### 3. Tracking & Resolution
- **Path**: Dashboard → Requests → Track
- **Button**: "Resolve Request"
- **Purpose**: To update the status and finally close out the administrative ticket.
- **Actions**:
  - Input: Resolution details, attachment (receipt/proof).
  - Monitoring: Real-time progress bar.
  - Trigger: "Update / Resolve"
  - Result: Requester notified of completion, audit log archived.

## Core Integrations
- **All Modules**: Centralized intake point for any internal support needs.
- **Procurement**: Triggering supply orders from administrative requests.
- **Finance**: Reimbursement tracking for travel or facility expenses.
